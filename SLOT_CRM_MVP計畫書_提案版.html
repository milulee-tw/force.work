Chapter 01
核心設計：碎片集卡 × 雙軌積分解鎖

1.1 為什麼選擇這個設計方向

SLOT 遊戲 CRM 的核心問題是：如何讓玩家每天有一個具體的、小的理由回來。廣告和現金補貼只能帶來短期流量，真正有效的留存來自遊戲內建立「每天有事要做、有東西在進行中」的目標感。

本計畫以「碎片集卡 + 解鎖積分」作為 MVP 核心機制：碎片讓玩家每天登入都有進展，解鎖積分讓押注行為直接對應道具卡的獲得，兩者共同形成每日回訪的內在動力。

碎片是道具卡的鎖定預覽。玩家持有它、看得見它，知道「我已經有這張卡，只差解鎖」。這種擁有感比等待感更能驅動行動。

1.2 雙軌積分：完全獨立的兩個數值

押注行為同時給予兩個完全獨立的積分，各自服務不同目的，互不干擾。

Chapter 02
EXP 架構：解鎖積分 × VIP 積分

2.1 解鎖積分：消耗對照表

解鎖積分的消耗量依道具卡星級設計，確保玩家必須真實押注才能取得對應價值的道具卡。特殊遊戲卡因 Free Game 期望值約為旋轉卡的 100 倍，解鎖成本也相應提高約 100 倍。

碎片門檻設定說明

旋轉卡碎片門檻設為 3 張（非 5 張），因為解鎖積分已是主要的防套利機制。光靠登入碎片湊到 3 張，沒有解鎖積分仍無法解鎖。較低的碎片門檻讓玩家更快看到解鎖清單，增加選擇感與期待感，而財務控制的責任由解鎖積分承擔。

2.2 VIP 積分與等級系統

VIP 積分永久累積，是玩家歷史押注總量的可視化表現。VIP 等級決定每日登入碎片的星級，高 VIP 玩家每天登入就能獲得更高星的碎片——這是最直接可感知的 VIP 特權差距。

解鎖積分倍率的意義

VIP 5 玩家押注相同金額，賺到的解鎖積分是普通玩家的 3 倍，意味著鯨魚可以更頻繁解鎖高星道具卡。這讓「押得越多、回報越快」的感知直接可量化，而不只是「等級更高」的空洞特權。VIP 積分倍率維持 ×1.0 不變，確保 VIP 等級只和真實累積押注量掛鉤。

3.2 碎片來源：兩種獨立管道

來源一　每日登入
星級由 VIP 等級決定
獎勵忠誠度。VIP 越高，每天登入拿越高星碎片。

MVP：VIP 系統未上線，全員固定 1星碎片 ×1。後端讀 vip_level，null 時預設 1星。Phase 3 VIP 上線後自動對應，發碎片邏輯不需修改。
來源二　每日流水任務
星級由當日 bet 段決定
獎勵當日活躍度。當天押得越高，達標後拿越高星碎片。

登入碎片與流水碎片同星級共同累積，推進同一條進度條。當天登入又達標的玩家可拿到 2 張同星碎片，進度是只登入的兩倍。

3.3 道具卡到期設計

碎片鎖定狀態永久保留，無到期壓力，玩家可以慢慢累積。玩家選擇解鎖後道具卡進入可用區，起算 14 天。背包平時只顯示總數量，不顯示個別到期日。只在最近 3 天有卡即將到期時，才顯示「N 張旋轉卡將於 N 天內到期」的警示，每張卡只推一次通知。使用時後端自動取最早到期的那張（FIFO），玩家不需選擇。到期靜默移除，不主動發通知避免負面情緒。

Chapter 04
4 Sprint MVP 計畫（8 週）

每個 Sprint 結束時玩家都有完整可運作的體驗，不存在功能斷點。S1 結束能登入領碎片；S2 結束能完成流水任務、雙軌積分開始累積；S3 結束能手動選擇解鎖道具卡並在遊戲中使用；S4 是數據校準與決策版本。

S1 · 第 1–2 週
登入碎片 + 背包地基 + Tag 引擎
目標：Day 1 就有碎片可拿，背包看得到鎖定預覽
簽到整合進任務頁，「今日登入」= 第一個任務，完成給 1星碎片 ×1（鎖定狀態顯示在背包）
背包頁建殼：碎片鎖定區（N/3 進度條）+ 解鎖清單區 + 可用道具卡區 + 解鎖積分餘額顯示，S3 填解鎖互動
Tag 引擎後端：新玩家（≤7天）、高黏著（近7天≥5次）、低黏著（≤3次）、斷簽風險（≥2天）、斷簽回歸（≥3天重登）
斷簽回歸觸發：登入當下補發碎片 ×1 + 解鎖積分補償 + 彈窗「你的碎片還在，繼續集吧」
帳本結構預留所有欄位：vip_points_total、unlock_points_available、vip_level（null 預設 0）
後端重點（10天）雙軌帳本結構 + Tag 引擎 + 碎片帳本。前端（7天）任務頁框架 + 背包殼含積分顯示。
S2 · 第 3–4 週
流水任務 + 雙軌積分引擎 + RTP 地基
目標：押注行為開始同時累積解鎖積分和 VIP 積分
每日流水任務：達標給對應 bet 段星級碎片 ×1；鯨魚達高門檻直接給旋轉卡 ×1（跳過集卡，獎勵即時感知）
雙軌積分引擎：每筆押注同時計算 vip_earn_rate 和 unlock_earn_rate，各自累積，同一 transaction 確保原子性
card_type_config 參數表上線：各市場 bet 段、旋轉卡面值、解鎖積分門檻、碎片門檻由運營後台設定
RTP 撥出引擎（純後端）：每筆注單記錄基礎 RTP、撥出比例、即時 RTP，初版撥出 0.3%，硬下限寫死 server 端
低黏著 Tag 玩家流水門檻自動降 20%；MVP 階段解鎖積分賺取倍率全員 ×1.0
後端重點（14天）雙軌積分引擎 + RTP 引擎。數據（3天）積分水位看板 + RTP 看板。S1 前務必預設計資料模型。
S3 · 第 5–6 週
解鎖互動 + 背包正式上線 + 遊戲效果接入
目標：玩家第一次主動選擇解鎖，走完完整閉環
解鎖清單：碎片達 3 張的卡進入解鎖區，顯示「需 XXX 解鎖積分」與玩家目前積分餘額對比
玩家點「解鎖」→ 確認彈窗（消耗 XXX 積分）→ 確認後立即解鎖，碎片歸零，旋轉卡進可用區，通知推送
積分不足時：顯示「還需 XXX 積分，繼續押注即可獲得」，給玩家明確的行動目標，而非模糊等待
背包三區塊正式完整：鎖定碎片區 / 解鎖清單 / 可用道具卡，到期警示（剩3天才顯示，每張卡只推一次）
Unity 接入旋轉卡遊戲內效果（本 Sprint 唯一碰 Unity，S3 啟動前確認排程）
Unity 風險（6天）旋轉卡遊戲內效果接入，S3 前確認排程。解鎖操作需 row-level lock，後端需求文件要明確寫清楚。
S4 · 第 7–8 週
數據校準 + 召回強化 + 決策報告
目標：不加新功能，根據數據修正，輸出決策依據
校準碎片門檻與解鎖積分消耗：目標「活躍玩家 Day 5–8 首次解鎖旋轉卡」，觀察碎片達門檻後到解鎖的等待時間
若大量玩家積分不足無法解鎖：調低解鎖積分門檻，或提高積分賺取倍率，確保一般活躍玩家在合理時間內完成首次解鎖
強化斷簽召回：若斷簽回歸玩家當週留存 <25%，升級為補 2 張碎片 + 解鎖積分補償（約等於 1 天正常押注量）
決策報告：次日/7日留存變化、任務完成率、碎片首次解鎖完成率（最核心指標）、積分消耗分布、各 Tag 玩家行為差異
決策門檻首次解鎖完成率 >40% + 7日留存提升 >10% → 推薦 M3 引入 VIP 擴充或第二貨幣。

Chapter 05
第二貨幣：為什麼 MVP 不做，何時才考慮

5.1 解鎖積分與第二貨幣的關係

本計畫的解鎖積分在工程架構上已經是一種「有限用途的第二貨幣」——押注賺取，消耗解鎖道具卡。與完整第二貨幣的差異只在消耗場景的豐富度：解鎖積分目前只能解鎖道具卡，而第二貨幣可以用在更多場景（抽寶箱、幸運輪、排行榜兌換等）。

升級到第二貨幣不需要重構帳本，只需要把「解鎖積分」的消耗場景擴充。後端架構的地基 MVP 期間已建好，未來升級的工程成本很低。

5.2 第二貨幣的優缺點分析

優點
第二貨幣能額外解決什麼
統一所有 CRM 出口：簽到、任務、排行榜都發同一種幣，玩家只管一個餘額
可設計隨機消耗場景（抽寶箱、幸運輪），製造驚喜感與留存激勵
玩家有更豐富的選擇：決定幣要花在哪個消耗場景，主動感更強
餘額數字可視化，累積感比單一解鎖路徑更強
MVP 不納入的原因
現階段的複雜度與風險
並發問題：玩家主動兌換需要資料庫 row-level lock，MVP 已需要，但擴充場景後並發複雜度倍增
通膨風險：多個消耗場景需要「幣水庫」看板持續監控發行與消耗平衡
玩家學習成本：抽象貨幣比碎片集卡更難在 5 分鐘內建立直覺感知
運營複雜度：兌換率表、異常行為偵測、客訴處理都需要額外投入

5.3 從數據訊號決定是否引入

01
道具卡到期率 > 30%
玩家解鎖了太多卡用不完，但還在持續押注。代表需要更多消耗場景讓積分有地方花。
02
解鎖積分大量閒置
玩家積分餘額持續增長但不消耗，道具卡解鎖需求已飽和，代表消耗場景不夠豐富。
03
解鎖完成率曲線平坦
碎片達門檻後玩家長時間不解鎖。要先確認是積分門檻太高還是道具卡吸引力不足，再決定方向。
04
任務完成率無鋸齒形
正常情況碎片 2/3 那天任務完成率應明顯升高。若曲線平坦，代表集卡機制心理效果為零，先修任務設計。
引入第二貨幣的觸發條件

道具卡到期率 >30% 且 7日留存沒有同步下降。代表玩家仍活躍但解鎖機制的激勵上限已到，需要更多消耗場景。此時的工程升級成本很低——帳本不動，只擴充消耗場景。

5.4 無縫銜接路徑

MVP（現在）
unlock_points 帳本建立，唯一消耗場景是道具卡解鎖。
升級步驟 1
unlock_points 改名為 currency，新增發放來源（簽到、排行榜等）。
升級步驟 2
新增消耗場景（抽寶箱、幸運輪），加強並發控制，建幣水庫看板。
升級步驟 3
前台錢包 UI 上線，顯示餘額與所有消耗場景入口，碎片解鎖機制保留並行。

Day 5 回歸的設計關鍵

回歸時碎片已滿 3/3 直接進解鎖清單，玩家感受到「我已經可以解鎖了，只差積分」而非「我還要等流水達標」。補發的解鎖積分讓玩家在 Day 6 達到流水任務時的積分能夠解鎖，Day 7 完成解鎖閉環——從斷簽到第一張旋轉卡，整個過程在 3 天內完成，召回效果最大化。

6.2 偶爾回流小客（7–30 天）

時間點Tag 狀態系統回應設計目的
正常回來的那天低黏著登入碎片入帳（鎖定狀態），流水門檻自動降 20%讓每次回來都有進展，降低達標難度
碎片達門檻但積分不足低黏著背包顯示「還需 XX 積分解鎖，繼續押注即可」明確的行動目標，消除模糊等待感
積分足夠可以解鎖時低黏著背包解鎖清單高亮「可解鎖！」，輕推玩家行動解鎖是情感峰值，讓玩家主動感受選擇的快感
斷簽 3 天後回來斷簽回歸補發碎片 ×1 + 解鎖積分補償 + 彈窗「你的碎片還在」消除「之前白做了」的心理阻力
道具卡快到期低黏著剩 3 天內才顯示警示，每張卡只推一次通知在有意義的時間點提醒，不製造持續焦慮

6.3 鯨魚玩家（高 bet，VIP 5）

時間點Tag 狀態系統回應設計目的
每日高流水押注高黏著流水達高門檻直接給旋轉卡 ×1（跳過集卡）；解鎖積分賺取倍率 ×3.0（Phase 3 VIP 5 上線後）鯨魚不需集卡心理機制，直接給高價值獎勵且速度遠快於小客

Chapter 08
執行原則
1

碎片與背包同天上線

S1 碎片帳本和背包殼必須同日上線。玩家第一天拿到碎片就要有地方看到它的鎖定狀態和積分進度，不能有空窗期。

2

雙軌積分絕對獨立

vip_points_total 和 unlock_points_available 在同一個 transaction 更新，但邏輯完全分開。解鎖道具卡只扣解鎖積分，VIP 等級永遠不因使用獎勵而退步。

3

RTP 下限 Server 鎖定

S2 上線前 RTP 硬下限必須和財務確認，寫死後端，任何人工操作無法突破，保護財務安全底線。

4

bet_value 解鎖時寫死

道具卡面值在玩家選擇解鎖當下從參數表讀取並寫入卡片記錄，後台調整不影響已解鎖的卡，避免財務爭議。

5

S4 不加功能

任何 S4 期間的新想法一律進 Backlog。S4 只做校準和報告，讓數據說話，不以直覺加功能。

6

帳本預留所有擴充欄位

vip_level、free_game_bet、unlock_earn_rate、login_fragment_star 等 Phase 3 欄位在 S1–S2 即建立，MVP 填預設值，Phase 3 填值不改表結構。
