01
精品款 vs. 一般款：定義與標竿分析
Definition & Benchmark Analysis
核心命題：精品款的核心在於「情緒價值的極致堆疊」，而一般款僅在於「功能性的實現」。兩者的本質差距不在美術品質，而在於是否有「演出思維」貫穿全流程。
維度一般款（Factory Model）精品款（Boutique Model）標竿案例 & 具體觀察
設計邏輯
  • 套版思維，更換美術皮膚（Reskin）
  • 核心機制與流程不變
  • 美術與系統各自獨立設計
  • 敘事思維，所有動態與數值皆為主題服務
  • 系統事件（Win / Free Game）觸發演出節奏
  • 美術、音效、程式三軸同步設計
 Peter & Sons：
  • 強烈個人風格化，視覺語言高度一致
  • 每個畫面細節均為世界觀服務，無「通用元素」
互動體感
  • 點對點反應：Spin → 轉軸 → 結束
  • 玩家動作與遊戲回應之間無情緒鋪墊
  • 連鎖反應：Spin → 音效預熱 → 重量感旋轉 → 符號落地有衝擊
  • 每個動作均引發多層次視聽反饋
 PG Soft：UI 順滑度、數值跳動帶壓迫感

Peter & Sons：物件互動符合物理直覺，反饋感強烈
視覺權重
  • 靜態美，原畫品質高但動態生硬
  • 物件缺乏重量感、慣性感
  • 特效疊加頂層，與環境脫節
  • 動態美，物件間有光影影響與空間感
  • 消除時有餘震與回彈，強化「重量」感
  • 鏡頭有景深與運鏡，不是死板平視
 Peter & Sons：
  • 動態節奏極快但視覺層級分明
  • 前中後景分離清晰，視差層次豐富
品牌識別
  • 視覺風格偏向市場平均值
  • Operator 難以區分供應商差異
  • 第一眼即可辨認供應商風格
  • Operator 願意為獨家視覺支付更高分潤
 Peter & Sons：
  • 高度個人化的視覺語言，即使在 Lobby 縮圖中也能一眼辨識
  • Operator 主動以「風格獨特性」為賣點，玩家對供應商品牌有認知
02
現狀診斷：視聽體感完整對照表
Current State Diagnosis — Full Audio-Visual Assessment
診斷前提：CMB 目前處於「高品質素材、低度整合」的階段。問題不在於個別資源的品質，而在於「跨維度整合能力」與「演出思維」的系統性缺失。
視聽維度CMB 現狀症狀根本成因分析精品標竿優化執行動作驗收標準（AC）
美術風格
  • 圖騰材質各自獨立設計，混搭感強
  • 特效密度/大小與整體風格失衡
  • 缺乏統一環境光源基準
  • 無材質規範文件，外包各自定義
  • UI 設計師與背景設計師未跨部門對齊
  • 無光源參考板
  • 全域材質一致性
  • UI 質感環境化
  • 所有物件受同一虛擬光源影響
  • 建立材質庫（≤3 種核心材質）
  • 製作光源參考板，外包強制遵循
  • 圖騰辨識度以顏色分級規範
  • 5人盲測辨識度 ≥80%「風格統一」
  • 製作人 Sign-off 對比截圖
動畫細節
  • 動態曲線線性，缺乏加速/減速節奏
  • 符號落地無擠壓拉伸
  • 停輪無物理回彈，呈現僵硬停止
  • Spine 動畫由外包直接輸出，未經 TA 二次調整
  • 無動畫規範文件（Easing Curve 標準）
  • 缺乏物理知識人員審核
  • 符合物理律：擠壓/拉伸/回彈
  • 停輪有物理回彈效果
  • 環境光隨事件動態投射
  • TA 建立 Easing Curve 規範庫
  • 所有 Spine 原檔由 TA 審核後再交程式
  • 實作環境光遮蔽與光影聯動
  • 擠壓幀數 ≥3f，形變幅度 10-15%
  • Easing Curve 偏差 <5%
UI 體驗
  • 按鈕缺乏點擊反饋（無縮放/閃光/震動等即時回應）
  • 彈出面板/視窗缺少「來源感」，出現方式突兀
  • 面板展開/收起為直接顯示隱藏，無彈跳感
  • 共用 UI 元件套用通用樣式，未融入各遊戲主題視覺
  • UI 互動回饋規範缺失，點擊反應由程式自行決定
  • 彈出元件無出現動線設計（缺少 origin point 定義）
  • 面板展開/收起動畫未定義緩動曲線
  • 共用 UI 模組化設計優先，主題化適配未納入流程
  • 每次點擊有即時視覺/物理反饋（縮放/光效/震動）
  • 彈出面板從觸發來源位置展開，有方向感與邏輯感
  • 面板展開收起具彈跳緩動，符合物理直覺
  • 共用 UI 在保持功能一致性的前提下，視覺套入各遊戲主題
  • 企劃定義各類按鈕的點擊反饋規格（縮放幅度、持續時間）
  • 制定彈出元件的出現起點規範（origin 位置設計）
  • 建立展開收起 Easing Curve 標準（含彈跳係數定義）
  • 建立共用 UI 主題化適配 Checklist，各遊戲上線前逐項確認
  • 5人測試：點擊任意主要按鈕均感受到即時反饋
  • 彈出來源感：截圖可直觀判斷面板從哪裡「長出來」
  • 共用 UI 通過主題適配審核，不出現突兀通用元件
音效演出
  • 音效與動畫特效配合度尚可，但整合層次仍不夠細緻
  • 音效師為外包成員，難以深度參與演出節奏討論
  • 相似情境的音效設計缺乏明確差異化（如不同場景的觸發音效感覺接近）
  • 音畫同步有時存在微小落差
  • 音效師非 in-house 成員，無法全程深度介入演出設計
  • 缺少「盤面狀態 → 音軌層級」的完整對應規格文件
  • 音效整合以交付物驗收為主，缺乏逐幀校準的 AV Sync QA 流程
  • 音效情緒分級未與企劃的情緒曲線對齊
  • 音效層次與盤面狀態精準對位
  • Free Game 有明顯張力升級感
  • Impact Point 精準對位視覺峰值
  • 不同等級事件有清晰可辨的音效差異
  • 企劃輸出完整「情緒曲線 → 音效強度分級」對應表，作為音效師設計依據
  • 建立 AV Sync QA 流程（逐幀校準），作為驗收標準
  • 關鍵演出節點（Big Win / Free Game Entry）納入高階製作人深度審核
  • 音畫 Sync 誤差 <10ms（儀器量測）
  • 5人聽感：Free Game 張力感 ≥4/5
  • 各事件等級音效：5人盲測可正確分級辨識 ≥80%
鏡頭切換
  • 場景切換採硬切或黑屏
  • 演出動畫全屏遮蔽盤面，打斷心流
  • 缺乏前中後景分離的景深感
  • 無鏡頭設計規範，場景切換由程式自行決定
  • 演出動畫未考慮心流連續性
  • 背景為單層靜態圖，無視差設計
  • 無縫敘事，Zoom-in/Out 完成切換
  • 演出動畫不全屏遮蔽
  • 至少 3 層視差（前/中/後景）
  • 制定動態分鏡 (Storyboard)，禁止硬切
  • 場景切換採 mg fade out 同時 fg fade in
  • 背景重構為多層視差 Parallax
  • 轉場時間 ≤2 秒
  • 截圖可辨識 ≥3 層景深
  • 5人測試無「演出打斷感」反饋

🔍 診斷核心結論

五大維度的問題根源可以收斂到同一個系統性缺失：「缺乏跨職能的演出整合機制」

  • 外包交付物無規範，TA 缺位導致無人二次加工
  • 企劃未定義音畫 Sync 標準，音效師與動畫師各自開發
  • UI 工程師依工程邏輯決定跳錢曲線，非由企劃定義情緒節奏
  • 鏡頭設計無人主導，場景切換由程式自行判斷

結論：問題不是「人做不好」，而是「沒有人負責整合」。精品化的核心投資應放在建立整合機制與引進整合角色（TA + 演出企劃），而非增加美術人力。

03
精品化開發流程手冊（含 Gate Review 機制）
Boutique Development Framework with Gate Reviews
流程設計原則：打破「企劃→美術→程式」的線性交棒，改為「以演出為核心的迭代閉環」。每個階段設有明確的入場門票（Entry Criteria）與出場審查（Gate Review），未通過 Gate 須退回重做，不得強行進入下一階段。

📋 各階段細節
Phase 01
概念期
Concept & Pre-viz
▾
🎯 階段目標

確立遊戲的「情緒核心」，在任何美術資源投入之前，先以動態分鏡鎖定演出節奏。

📋 Entry Criteria（入場門票）
  • 主題世界觀文件已由高階製作人核准
  • 體驗向數學模型確認：FG 觸發頻率、倍率起伏節奏等玩家情緒體感相關參數已定義（RTP 數值由數學師負責，此階段僅確認「高點在哪裡、會不會太沉悶」的體驗向描述）
📦 核心產出
  • 情緒曲線圖（Emotion Arc）：定義 Normal Game / Big Win / Free Game 的情緒強度與節奏節點
  • 動態分鏡腳本（Storyboard）：逐幀標示演出時機、鏡頭運動、音效進入點
  • 視覺語言參考板（Mood Board）：含材質規範、調色盤、光源方向，確認 H1~H6 圖騰顏色分級應用於本款遊戲的配置
👥 職能分工
  • 高階製作人（主導 + 最終負責）：確認情緒曲線方向與 Storyboard 節奏，提供演出意圖的核心判斷，執行 Gate Review Sign-off
  • 資深企劃（執行）：撰寫 Storyboard 細節、產出 Mood Board、對齊 H1~H6 圖騰視覺規範在本款的應用
  • 程式 Lead（諮詢）：確認演出構想的技術可行性
  • 音效師（諮詢）：提前了解音效結構需求，確認分層可行性
  • 外包美術（知會）：提前接收風格規範與材質規範
🔒 Gate Review — 全部達成才可進入 Phase 2
  • 情緒曲線圖由高階製作人 Sign-off
  • Storyboard 完成率 ≥80%，主要演出節點（Spin / Stop / Win / Free Game Entry / Exit）全數覆蓋
  • Mood Board 材質種類 ≤3 種，色盤已鎖定；H1~H6 圖騰顏色分級在本款遊戲的配置已確認
⚠ 常見卡關風險
  • 跳過 Storyboard 直接發稿給美術 → 強制退回，未完成 Storyboard 不得發出任何美術需求單
  • 體驗向數學模型確認與數學師意見不一致 → 需先對齊「FG 頻率與情緒節奏感」再進入此階段
Phase 02
原型期
Graybox & Prototype
▾
🎯 階段目標

在不投入任何精細美術的情況下，以幾何圖形確認 Spin/Stop/Win 三段的動態節奏「爽快感」。此階段以動感為優先，美感為次。

💡 延伸價值：此階段驗收通過的轉動動態與對獎動態，可沉澱為「通用動態模組」直接套用至一般款遊戲，有效提升全線遊戲的動態品質基線，不限精品款才能受益。

📋 Entry Criteria
  • Phase 1 Gate Review 已通過
  • 程式框架（Reel Engine）已搭建完成
📦 核心產出
  • 灰模原型（Graybox Build）：可實際點擊操作，Spin/Stop/Win 動態節奏完整呈現
  • Easing Curve 標準庫初版：定義所有物件的加速/減速曲線，含 Symbol Landing / Loop / Win 動態類型規範
  • 動態時間規範草案：兌獎 1s、聽牌 4s、聽牌延遲 0.2s 等核心時間軸定義
  • 通用動態模組候選清單：標記哪些動態節奏可直接提供一般款套用
  • 動態節奏評估報告：由高階製作人、資深企劃共同評分
👥 職能分工
  • 高階製作人（最終負責 + 體感審核）：判斷 Spin/Stop/Win 節奏是否達到「爽快感」標準
  • TA / 資深動畫師（執行）：建立 Easing Curve 規範庫、定義動態時間標準、評估模組化可行性
  • 程式（執行）：Graybox Build 實作
🔒 Gate Review
  • Spin/Stop/Win 動態節奏經高階製作人 Sign-off，確認「爽快感」達標
  • Easing Curve 標準庫已記錄，程式端確認可實作
  • 動態時間規範草案已確認（含 Turbo 模式加速版本）
  • 通用動態模組候選清單已產出，可供一般款後續套用
⚠ 常見卡關風險
  • 美術希望在灰模階段加入素材「先看效果」→ 嚴格禁止
  • 動態節奏未達標但時程壓力下強行通過 → 此為最常見的品質崩潰點，Gate 必須硬守
Phase 03
生產期
Iterative Production
▾
🎯 階段目標

以模組化方式生產精細美術資產，TA 全程把關整合品質，確保外包交付物符合精品標準後才進入程式整合。

📋 Entry Criteria
  • Phase 2 Gate Review 已通過
  • 材質庫與光源規範文件已發布至外包
  • 外包合約中已明確要求提交 Spine 原檔（非只交 Export 結果）
📦 核心產出
  • 模組化資產包：符號（含 Landing / Loop / Win / Idle 動態齊備）、特效、UI 元件，每項均附 TA 審核通過記錄
  • 光源整合完成報告：確認所有資產受統一虛擬光源影響
  • UI 動畫時間規範文件：面板淡入淡出 0.2s、場景轉換黑幕 0.1s 等完整規範
  • 音效分層規範執行記錄：確認盤面狀態→音軌層級對應表已實作
📌 外包管理關鍵規定
  • 外包交付需同時提交：Export 結果檔 + Spine 原檔 + 材質來源檔
  • TA 審核外包交付物，不符合 Easing Curve 規範或光源規範者退回
  • 退回次數超過 2 次，自動觸發 Lead Review
  • 特殊圖騰（SS / Wild）需有額外轉動期間提示特效設計，落地動態需在 1s 內完整播放
🔒 Gate Review
  • 所有核心資產（主視覺符號、UI 主框架、Free Game 背景）完成 TA 審核
  • 程式完成 50% 以上資產整合，無重大技術問題
  • Alpha Build 由演出企劃確認整體節奏無明顯偏離 Storyboard
Phase 04
打磨期
Polishing & Certification
▾
🎯 階段目標

針對完整 Build 進行逐項精品化檢核，未通過項目分級處理。Blocker 等級問題必須修復才可進入正式版本，不接受「先上線再改」。

📋 Entry Criteria
  • Phase 3 Gate Review 已通過
  • 完整 Beta Build 可正常遊玩，無 Critical Bug
📦 核心產出
  • 精品化檢核表完整填寫版（本文第四章，含精品標準 + 生產品質 SOP）
  • 音畫同步測試報告（含儀器量測數據）
  • 玩家感知測試報告（至少 5 人，含體感評分）
  • 視覺錯誤掃描報告（背景壓縮、圖層錯誤、UI 對齊等）
🔒 Final Release Gate
  • 所有 Blocker 等級項目 100% 通過
  • Major 等級項目通過率 ≥80%
  • 製作人最終 Sign-off
⚠ 常見卡關風險
  • 上市壓力導致跳過打磨期 → 打磨期應設為不可壓縮的硬性預留時間（建議 ≥2 週）
📌 流程回流機制：任何 Gate 未通過均退回當前 Phase 修正。退回同一問題 ≥2 次自動上升製作人進行根因分析。
回流路徑：Phase 4 → Phase 3（資產品質）　｜　Phase 4 → Phase 2（節奏根本問題）　｜　Phase 3 → Phase 1（風格根本偏離）

數據支撐與戰略結論
Data-Driven Conclusion & Strategic Direction
為什麼精品化是唯一路徑

消除「拼裝感」能降低玩家的認知負荷（Cognitive Load），讓玩家更容易進入「心流狀態（Flow）」。心流狀態的核心指標直接反映在 Average Bet Count（平均投注次數）的提升上。

可觀測的業務影響鏈

精品化降低視覺混亂感 → 玩家認知負荷降低

認知負荷降低 → 玩家更容易進入心流 → Average Session Length 延長

演出節奏精準對位數學節奏 → 中獎時情緒峰值更強烈 → Repeat Play 意願提升

整體品牌辨識度提升 → Operator 主動推廣意願增加 → 遊戲曝光量提升

核心投資優先順序
1
補齊 TA 或資深動畫師 — 技術整合人才的缺位是所有「拼裝感」的根本原因，是精品化最關鍵的單一投資
2
